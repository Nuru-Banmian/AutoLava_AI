# 每日记账日历记录标记设计

> 状态：待用户重新复核
> 日期：2026-07-18

## 目标

在每日记账的日期选择器中，用户浏览任意历史月份时，所有已有台账记录的日期都显示小蓝点，方便识别尚未录入的日期。选择已有记录日期后，页面继续自动回填该记录的数据，用户只需修改需要调整的字段。

## 方案

日期选择器将当前浏览月份通知每日记账页面。页面以门店 ID 和月份为粒度请求该月的记录日期，并由查询缓存复用已访问月份的数据。日历继续使用现有蓝点样式；仅在成功加载的数据中标记日期。

## 数据流

1. 打开日期选择器时，页面请求当前选中日期所在月份的记录日期。
2. 用户点击上一个月或下一个月时，日期选择器更新浏览月份；页面请求对应月份的记录日期。
3. 返回的记录日期转换为 `Set<string>` 并传给现有日历组件。
4. 保存台账成功后，失效或刷新该记录所属月份的查询，使该日期立即出现蓝点。
5. 切换门店时，查询键隔离不同门店的数据，不会混用标记。
6. 用户选择已有记录日期后，沿用现有的单日记录查询和表单回填：营业状态、收入项目或营业额、洗车数量、天气和活动均使用已保存数据作为初始值；保存操作继续为修改而非重新录入。

## 边界与错误处理

- 仅显示真实已保存的台账记录；未录入日期不显示标记。
- 月份数据加载中或请求失败时不显示推测性标记，日期选择仍可用。
- 现有“最近七天”、当前日期记录及补记/编辑提示保持不变。
- 日期标记查询只提供日历提示，不替换或延迟已有记录详情的单日查询与表单回填。
- 仅允许浏览到今天所在的月份，沿用现有限制。

## 测试

- 切换到不在最近七天内的历史月份后，日历显示该月已保存记录的蓝点。
- 不同门店和月份的标记数据互不串用。
- 保存当前月份的一条记录后，该月查询被刷新并在日历中显示蓝点。
- 选择蓝点日期后，表单会自动显示该记录的已保存字段；修改单个字段并保存时，其余字段保持原值。
- 既有日期禁用、键盘导航及已选日期样式继续通过。
