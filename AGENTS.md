# 项目开发节奏

本项目默认采用快速迭代流程，除非用户明确要求更严格的流程：

1. 开发单个小任务时，只运行直接相关的测试、类型检查或构建检查。
2. 完成一组相关功能后，运行该功能所属前端或后端的完整测试。
3. 提交 PR、合并分支或用户明确要求发布前，统一运行前端全量测试、生产构建、后端全量测试；涉及关键交互时再运行端到端测试。
4. 代码审查集中在一组功能完成后，不对每个小改动重复进行多轮完整审查。
5. 本地按功能提交；仅在阶段完成或用户明确要求时推送或更新 PR。
6. 低风险的文案、布局和样式调整，不重复运行无关的后端测试。

保留必要的质量门槛：发现失败、回归或高风险变更时，优先补充或运行针对性的测试，并在 PR 前完成统一验证。
